Stereo Needle Keypoint and 6-DoF Pose Estimation in Surgical Video

Method description — multi-class segmentation, rule-based stereo keypoint extraction, a learnable heatmap detector, and rigid-arc pose recovery.

1. Overview

Given a calibrated binocular endoscopic stream, we estimate N equally spaced keypoints along the visible needle centerline (tip→tail) and the rigid 6-DoF pose of the needle. A semantic segmentation network provides per-pixel class masks; a geometric module then reconstructs the needle as a planar circular arc, samples keypoints by arc length, triangulates them across the stereo pair, and recovers the pose. A learnable heatmap detector is trained on the geometric module's (human-verified) output to provide a mask-independent, real-time alternative.

3. Stereo Geometry

Pixels are undistorted with (K, D). A 2D point p is back-projected and a 3D point is obtained by linear triangulation of the corresponding rays:

X = Triangulate(P1, P2, pL, pR), solving pL × (P1X) = 0, pR × (P2X) = 0 (DLT).

All 3D quantities are metric (mm) since ‖t‖ is the physical baseline.

4. Per-view Needle Centerline

From the needle mask Mv=1 (view v∈{L,R}) the centerline is reconstructed as a single planar arc, robust to occlusion that fragments the mask:

  1. Collect skeleton pixels from all connected components of the needle mask.
  2. Robustly fit one ellipse ℰv (RANSAC/trimmed least squares), rejecting outlier blobs.
  3. Order inlier points by ellipse angle; remove the largest angular gap; resample the spanned arc to obtain the ordered, gap-filled centerline 𝒞v (occluded segments are interpolated along the arc).

Tip/tail orientation: the endpoint nearest the suture-thread mask (class 2) is the tail; the opposite endpoint is the tip. A swage-thickness fallback (distance transform) is used when the thread is not visible.

5. Equally-spaced Keypoints

Each view's centerline is resampled at equal arc-length fractions to give N ordered points (tip→tail):

piv = 𝒞v(ui),   ui = i/(N−1)·Lv,   i = 0,…,N−1, where Lv is the polyline length. Spacing is uniform along the centerline regardless of occlusion.

6. 3D Keypoints

The matched samples are triangulated by index: Xi = Triangulate(P1,P2, undistort(piL), undistort(piR)). A visibility flag vi∈{0,1} marks whether pi lies inside the observed mask (1) or on a reconstructed/occluded segment (0).

7. 6-DoF Pose (Rigid Planar Arc)

The needle is a rigid planar circular arc, so its pose is fully defined by the fitted 3D circle. Fitting {Xi} by SVD gives a plane (normal n), in-plane circle center o and radius ρ. The object frame is

ez = n (sign-fixed so nz<0, facing the camera),   ex = Πn(Xtip−o)/‖·‖,   ey = ez×ex,

Ro = [ex ey ez] ∈ SO(3),   to = o ∈ ℝ³,   output also as a rotation vector r = log(Ro) ∈ ℝ³ (Rodrigues),

where Πn projects onto the arc plane. A constant-velocity Kalman filter on (to, r) ∈ ℝ⁶ (12-state, predict/correct) temporally smooths the pose and coasts through frames without a detection.

8. Learnable Keypoint Detector

8.1 Architecture

The same DINOv2 (ViT) encoder + DPT dense head as the segmentation model, with the output layer producing K=N+1 heatmap channels: N single-peak needle channels (tip…tail, fixed identity) and one multi-peak claw-tip channel. Input I∈ℝ^(3×W₀×W₀) (resized to a multiple of the patch size 14); output 𝐇∈ℝ^(K×h×w), h=w=W₀/s.

8.2 Targets and Loss

For a labelled keypoint at (x,y) the target channel holds a Gaussian G(·)=exp(−‖·−(x,y)/s‖²/2σ²). Channels with no keypoint carry weight 0. Training minimizes a channel-weighted MSE:

ℒ = ( Σk wk·MSE(𝐇k, Gk) ) / Σk wk,   wk∈{0,1}.

8.4 Inference (hybrid)

Per view, peaks are decoded from 𝐇 (arg-max for needle channels; threshold + non-max suppression for the claw channel). Needle keypoints correspond across views by channel identity and are triangulated (§6); the 6-DoF pose follows (§7). The detector is mask-free at run time; the geometric module remains the auto-labeler and the 3D/pose refiner.

9. Real-time Pipeline and Metrics

Each stereo frame is segmented (both eyes in one batched fp16 forward), processed to keypoints + pose, and displayed with the instantaneous frame rate. Accuracy is reported against the (verified) reference sidecars: per-keypoint left-view pixel error, PCK@τ (fraction with error ≤ τ px), and 3D error (mm) when 3D references exist.

Limitations. The stereo baseline is small, so the fitted radius and plane normal — hence absolute 3D/pose magnitude — are sensitive; argmax keypoint locations and the temporally filtered pose remain stable. Rule-based claw localization is bounded by mask coverage of the thin reflective jaw tips; the learnable detector mitigates this once trained on corrected labels.
